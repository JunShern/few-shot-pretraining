    "text": "Data-BitStream-0.07\n\nNAME\n\nVERSION\n\nversion 0.03\n\nDESCRIPTION\n\nA role written for Data::BitStream that provides get and set methods for Gamma-Golomb codes. The role applies to a stream object.\n\nGamma-Golomb codes are basically Golomb codes using the Elias Gamma code for the quotient instead of a Unary code. This makes them suitable for occasional large outliers that would otherwise use thousands or millions of bits to encode.\n\nIn particular, the GammaGolomb(3) code is interesting for some distributions.\n\nMETHODS\n\nProvided Object Methods\n\nput_gammagolomb($m, $value)\n\nput_gammagolomb($m, @values)\n\nInsert one or more values as Gamma-Golomb codes with parameter m. Returns 1.\n\nget_gammagolomb($m)\n\nget_gammagolomb($m, $count)\n\nDecode one or more Gamma-Golomb codes from the stream. If count is omitted, one value will be read. If count is negative, values will be read until the end of the stream is reached. In scalar context it returns the last code read; in array context it returns an array of all codes read.\n\nParameters\n\nThe parameter m must be an integer greater than or equal to 1.\n\nThe quotient of value / m is encoded using an Elias Gamma code, followed by the remainder in truncated binary form.\n\nNote: if m == 1 then the result will be coded purely using gamma coding.\n\nNote: if m is a power of 2 (m = 2^k for some non-negative integer k), then the result is equal to the simpler ExpGolomb(k) code, where the operations devolve into a shift and mask."
